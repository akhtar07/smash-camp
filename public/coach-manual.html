Logging In & Logging Out

Logging In

  1. Go to smash-camp.vercel.app
  2. Enter your Email and Password on the Login tab.
  3. Tap Login — you go straight to the Coach Dashboard.

Logging Out

  1. From any screen, tap the logout icon (arrow pointing right) in the top-right corner of the header bar.
  2. You will be returned to the login screen.

💡 Stay logged in: The app keeps you logged in between sessions. You don't need to log in every time you open the app, as long as you don't tap logout.

Monitoring Today's Attendance

Players log their own attendance by tapping "I'm Here" on their Dashboard. As coach you can monitor this in real time.

How to Check Who Is Present

  1. Open your Coach Dashboard.
  2. Look at the stats grid at the top — it shows how many are present vs absent right now.
  3. Look at the Roll Call section — every player's avatar shows a green dot if present, grey if absent.
  4. To see a player's full attendance history, tap their avatar or their name in the roster list.

📋 Note for coaches: You cannot manually mark a player as present from the coach interface. Players must mark their own attendance on their own device using the "I'm Here" button. If a player forgot to mark attendance, ask them to open the app and do it themselves — they can mark it any time on the same day.

Viewing Individual Player Stats

You can tap any player (from the Dashboard roster, Squad page, or Roll Call) to see their detailed stats page.

What the Player Detail Page Shows

  1. A header with the player's name and a PRESENT / ABSENT badge for today.
  2. Three stat cards: Attendance % (days present / camp days so far), Avg Completion % (average across all sessions), and Camp Day counter.
  3. A complete Session History list showing every session the player has logged, with the date and their completion % for that day. Color coded: green = 80%+, blue = 50–79%, red = below 50%.

How to Access a Player's Page

  1. From the Coach Dashboard, tap any player row in the Squad Roster list.
  2. OR from the Squad tab, tap any player card.
  3. OR from the Roll Call section on the dashboard, tap any avatar bubble.
  4. To go back, tap the ← arrow in the top-left corner.

Squad Page — Full Roster Overview

Tap Squad in the bottom nav to see the full social view of all players.

What the Squad Page Shows

  • 3 stats at the top: Total Players, Here Today, Top Streak
  • "Training Today" section: avatar circles for players who are present right now
  • Full Roster list: ranked by score, showing streak 🔥, days attended, avg completion

As a coach, tapping any player card takes you to their individual detail page. Players cannot tap each other's cards (they see stats but cannot navigate to detail pages — that is coach-only).

Calendar Page — Attendance History

Tap Calendar in the bottom nav to see your own attendance calendar (as a coach your personal attendance is tracked here).

💡 Note: The Calendar page shows the logged-in user's attendance. When a coach views it, it shows the coach's own attendance (if they marked it). To view a player's attendance calendar, use the Player Detail page instead.

Reading the Calendar

  • Green cell with dot = Present that day
  • Dark cell = Absent (camp day that has passed)
  • Bright green cell = Today
  • Faded cell = Future day

Use the ← → arrows at the top of the calendar to navigate between months. The stats at the top show: Attendance %, days present, days left in camp, and current camp day number.

Leaderboard & Scoring System

Tap Scores in the bottom nav to see the full camp rankings.

How Scores Are Calculated

ComponentWeightHow It's Measured
Attendance %50%Days present ÷ Total camp days so far × 100
Average Completion %50%Average of all session completion %s logged
Final Score100(Attendance% × 0.5) + (AvgCompletion% × 0.5)

✅ As a coach: To motivate players, remind them that showing up AND completing exercises are equally important. A player who attends every day but doesn't complete exercises will score the same as a player who never shows up but completes 100% when they do attend.

How to Add or Edit Exercises in Any Day

The training program for all 45 days is stored in a single file in the GitHub repository. You can edit this file directly on GitHub — no coding experience required. Changes deploy automatically to the live app within 1–2 minutes.

📁 File location: The file you need to edit is at src/lib/program.ts in the GitHub repository at github.com/akhtar07/smash-camp

Step 1 — Open the File on GitHub

  1. Open a browser and go to github.com/akhtar07/smash-camp
  2. You need to be logged in to a GitHub account that has write access to this repository. Ask the developer to add you as a collaborator if needed.
  3. Click the src folder → then lib folder → then click the file program.ts
  4. You will see the full program file. Click the pencil icon ✏️ in the top-right area of the file to edit it.

Step 2 — Understanding the File Structure

Each exercise follows this exact format:

// Each exercise looks like this:
{
  name: "Push-ups",
  sets: "3",
  reps: "12 reps",
  rest: "60s",
  rpe: "7",
  cue: "Keep your body straight and elbows at 45 degrees.",
  url: "https://www.youtube.com/results?search_query=push+up+form"
}

Each field means:

FieldWhat to WriteExample
nameThe exercise name exactly as you want it displayed"Goblet squat"
setsNumber of sets as a text"3" or "4"
repsReps or duration"10 reps" or "45 sec"
restRest time between sets"90s" or "-" for none
rpeEffort level from 1–10"7" or "6-8"
cueCoaching instruction for the player"Drive through your heels."
urlYouTube search link (optional)"https://www.youtube.com/results?search_query=..."

Step 3 — How Days Are Organised in the File

Days 1, 2, and 3 have their own unique exercise blocks defined as day1, day2, and day3 near the top of the file. Days 4–45 use shared default blocks (Foundation/Build/Peak/Taper phases).

A full day block looks like this:

// Example: How Day 2 is structured
const day2: DayProgram = {
  day: 2, phase: "Foundation", color: "emerald",
  blocks: [
    {
      block: "Warm-up",
      exercises: [
        { name: "Skipping rope", sets: "3", reps: "1 min",
          rest: "30s", rpe: "4", cue: "Stay light on feet.",
          url: "https://www.youtube.com/results?search_query=skipping+rope" }
      ]
    },
    {
      block: "Main",
      exercises: [
        { name: "Goblet squat", sets: "3", reps: "10",
          rest: "90s", rpe: "6-7", cue: "Knees track over toes.",
          url: "https://www.youtube.com/results?search_query=goblet+squat" }
      ]
    },
    {
      block: "Cool-down",
      exercises: [
        { name: "Couch stretch", sets: "2", reps: "45s",
          rest: "-", rpe: "2", cue: "Relax and breathe.",
          url: "https://www.youtube.com/results?search_query=couch+stretch" }
      ]
    }
  ]
};

Step 4 — Adding a New Exercise to an Existing Day

  1. Find the day you want to edit. For Day 1, look for const day1. For Day 2, look for const day2.
  2. Find the block you want to add to — "Warm-up", "Main", or "Cool-down".
  3. Inside that block's exercises: [...] array, go to the end of the last exercise entry (after its closing }) and add a comma.
  4. Then type your new exercise in the format shown above.
// Example: Adding a new exercise to Day 2 Main block
{
  block: "Main",
  exercises: [
    { name: "Goblet squat", ... },   ← existing exercise
    { name: "Romanian deadlift", ... }, ← existing exercise
    { name: "Box jump", sets: "3", reps: "8 reps",
      rest: "90s", rpe: "8",
      cue: "Land softly, absorb with knees bent.",
      url: "https://www.youtube.com/results?search_query=box+jump+tutorial" }
    ↑ new exercise added at the end
  ]
}

Step 5 — Adding a New Exercise to Days 4–45 (Shared Blocks)

Days 4 through 45 use shared exercise blocks at the top of the file called sharedExercises. To add an exercise that appears on ALL those days, edit the sharedExercises block near the top of the file following the same format.

⚠️ Important: Editing sharedExercises changes ALL days from 4 to 45 at once. If you only want to change one specific day, you need to give that day its own block (like day1, day2, day3). Ask the developer if you need help with this.

Step 6 — Saving and Deploying

  1. After making your changes in the GitHub editor, scroll down to the bottom of the page.
  2. You will see a section called "Commit changes".
  3. In the first text box, write a short description of what you changed. For example: "Added box jump to Day 2 main block"
  4. Leave the option as "Commit directly to the master branch".
  5. Click the green "Commit changes" button.
  6. Vercel (the hosting service) will automatically detect the change and redeploy the app. This takes 1–2 minutes.
  7. After 2 minutes, refresh smash-camp.vercel.app and open the session for that day — your new exercise will appear.

✅ No technical knowledge required. As long as you copy the format exactly (curly braces, commas, quotes), the change will work. If you make a mistake, the app will simply not deploy and the old version stays live — nothing breaks permanently.

Quick Reference — YouTube URL Format

To add a YouTube search link for any exercise, use this URL format:

https://www.youtube.com/results?search_query=exercise+name+here

Examples:
https://www.youtube.com/results?search_query=box+jump+tutorial
https://www.youtube.com/results?search_query=badminton+footwork+drill
https://www.youtube.com/results?search_query=lateral+lunge+exercise

Replace spaces in the exercise name with + signs.

Understanding the Database (Supabase)

All player data is stored in Supabase — a cloud database. You rarely need to access it directly, but here's what's stored and where.

Database Tables

Table NameWhat It Stores
playersEvery registered user — name, email, and whether they are a coach (is_coach = true/false)
attendanceOne row per player per day they marked present. Contains player_id, date, present=true
exercise_completionsEach exercise a player marks as done — player_id, day_number, exercise_name, completed, date
daily_sessionsSummary per session — player_id, day_number, date, completion_percent
nutrition_logsMeals and hydration logged per player per day

How to Access Supabase (if needed)

  1. Go to supabase.com and log in with the project owner's account.
  2. Open the smash-camp project.
  3. Go to Table Editor in the left sidebar to view and edit data directly.
  4. Go to Authentication → Users to see all registered users and manage accounts.

⚠️ Be careful: Deleting rows in Supabase is permanent. Only delete data if you are certain. Always double-check before clicking Delete in the database.

Troubleshooting Common Issues

Player Can't Log In

  1. Ask them to check they are using the correct email and password.
  2. If they forgot their password, go to Supabase → Authentication → Users, find their email, click the three dots → Send password reset.
  3. If their account is unconfirmed (unlikely with current settings), go to Authentication → Users → find them → click three dots → Confirm user.

Player Forgot to Mark Attendance

  1. Ask the player to open the app on the same day and tap "I'm Here" — they can do it any time before midnight.
  2. If the day has passed, you can manually insert an attendance record in Supabase: go to Table Editor → attendance → click Insert row → enter player_id (from the players table), date (format: YYYY-MM-DD), present = true.

App Not Updating After Code Change

  1. Wait 2–3 minutes after committing to GitHub.
  2. Hard-refresh the browser: on mobile, close the tab completely and reopen.
  3. Check Vercel deployment status at vercel.com — look for any failed deployments shown in red.

New Player Not Appearing in Squad/Dashboard

Players appear immediately after they sign up. Refresh the page (pull down to refresh on mobile) and they should appear. If not, check Supabase → Authentication → Users to confirm their account exists.

Scores Showing as 0 for a Player

This means the player has either not marked attendance or not completed any exercises. Their score will only update after they actively use the app. Remind them to mark attendance daily and complete exercises through the session page.
